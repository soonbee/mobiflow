# review-ui-app

모바일 앱(iOS/Android)에 특화된 UI 코드 리뷰 항목. `review-ui-common`의 공통 항목을 먼저 적용한 뒤, 본 스킬의 모바일 특화 항목을 추가로 점검한다.

**파일을 수정하지 않는다.** code-reviewer 에이전트의 Part A에 통합된다.

## Mobile-Specific Checklist

### 1. 화면 크기와 Safe Area

모바일 기기는 노치, 다이나믹 아일랜드, 홈 인디케이터, 폴더블 등 물리적 제약 존재.

- **Safe Area 적용**: 상단/하단 콘텐츠가 Safe Area를 존중하는가
- **다양한 화면 비율**: SE(4.7") ~ Pro Max(6.7") 등에서 레이아웃 정상. 고정 높이가 작은 기기에서 잘리지 않는지
- **폴더블/태블릿**: 폴더블의 hinge area, 태블릿의 넓은 화면 고려 (해당 시)
- **가로/세로 모드**: orientation 지원 시 전환 시 레이아웃 재배치
- **다이나믹 타입**: 시스템 글꼴 크기 키웠을 때 레이아웃 안 깨지는지. 고정 높이에 텍스트 잘리지 않는지

### 2. 네이티브 인터랙션 패턴

iOS와 Android 사용자는 각 플랫폼 패턴을 기대.

**iOS:**
- 좌→우 스와이프 뒤로 가기
- push/pop 네비게이션
- 바텀 시트, 액션 시트가 iOS 스타일
- 스크롤 바운스 자연스럽게
- 대형 타이틀 적절히 (해당 시)

**Android:**
- 시스템 백 버튼/제스처 동작
- Material Design 3 패턴 (FAB, Top App Bar, Bottom Navigation)
- 리플 효과
- Predictive Back (Android 14+)

**크로스 플랫폼:**
- 플랫폼별 UI 분기 (예: 날짜 선택 시 iOS=wheel, Android=calendar)
- 기본 컴포넌트가 플랫폼에 맞게 렌더링

### 3. 제스처 처리

여러 제스처가 동시에 존재. 우선순위 명확하지 않으면 의도와 다른 결과.

- **제스처 충돌**: 수평 스와이프 카드가 수직 스크롤 안에 있을 때 의도한 방향만 반응
- **제스처 우선순위**: 중첩 제스처의 우선순위가 명시적으로 설정
- **취소 가능한 제스처**: 삭제 스와이프 등 파괴적 동작에 취소 옵션 (되돌리기·확인 다이얼로그)
- **제스처 영역 크기**: 드래그 핸들·슬라이더 터치 영역 충분 (44pt × 44pt 이상)
- **네이티브 제스처와 공존**: 커스텀 제스처가 시스템 제스처(edge swipe back, 상태바 탭 등) 가로채지 않음

### 4. 리스트와 렌더링 성능

모바일은 메모리·CPU 제한적. 긴 리스트, 복잡 셀, 애니메이션에서 성능 문제 도드라짐.

- **리스트 가상화**: 긴 리스트에 가상화 적용 (FlatList, FlashList 등)
- **이미지 캐싱**: 네트워크 이미지에 캐싱 전략 (FastImage, expo-image 등)
- **오버드로**: 불투명 뷰가 겹쳐 같은 픽셀 여러 번 그리지 않는지
- **메인 스레드 부하**: 무거운 연산이 메인/UI 스레드에서 실행되지 않는지
- **애니메이션 프레임**: 네이티브 드라이버(`useNativeDriver`, Reanimated) 활용

### 5. 오프라인 및 불안정한 네트워크

- **네트워크 끊김 시 UI**: 빈 화면 대신 마지막 데이터 또는 명확한 오프라인 안내
- **재연결 시 동작**: 자동 갱신, 사용자 알림
- **느린 네트워크**: 3G 수준에서 타임아웃 적절. 무한 로딩 방지
- **낙관적 업데이트**: 좋아요·북마크 등 즉각 피드백이 필요한 동작에서 서버 응답 전 UI 반영

### 6. 플랫폼별 분기 — 깔끔한 조건 처리

- **분기 패턴 일관성**: `Platform.OS` / `Platform.select` / `.ios.js`/`.android.js` 분리 등 하나의 전략 일관 사용
- **분기 최소화**: 불필요한 분기 공통화
- **하드코딩 매직 넘버**: `Platform.OS === 'ios' ? 20 : 0` 흩어진 것을 상수 추출

### 7. 앱 생명주기

- **상태 복원**: 백그라운드 복귀 시 마지막 화면·입력 데이터 유지
- **타이머/인터벌**: 백그라운드 진입 시 정리, 복귀 시 재시작
- **음악/영상 재생**: 인터럽트(전화 등) 처리

## 출력 통합

본 스킬도 `review-ui-common`처럼 단독 출력 형식이 없다. `code-reviewer` 에이전트가 Part A 항목으로 통합 (area에 모바일 특화 카테고리 명시).

### Severity 가이드 (특화)

- **🔴**: Safe Area 미적용, 시스템 제스처 가로채기, 메인 스레드 블로킹, 메모리 문제로 크래시 가능
- **🟡**: 네이티브 패턴 불일치, 제스처 충돌, 오프라인 미처리, 이미지 캐싱 누락
- **pass**: 발견 사항 없음
